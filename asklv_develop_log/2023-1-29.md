### 尝试解析`.run`并且自行打包

使用命令`bash *.run --noexec --target XXX`意思是不运行安装的脚本，把*.run解压到123目录。
这里得到几个文件。

自行打包，脚本中提示了使用makeself这个工具。

下载：makeself

```bash
wget https://github.com/megastep/makeself/releases/download/release-2.4.5/makeself-2.4.5.run
bash makeself-2.4.5.run
```

设置环境变量，索引到`makeself.run`脚本，执行命令完成脚本的生成。

```bash
makeself.sh ejabberd-22.10-1-linux-x64-src/ ejabberd-22.10-1-linux-x64-testrun.run "Asklv Test package" ./setup
```

标准做法
```bash
makeself.sh [args] archive_dir file_name label startup_script [script_args]
```

> 注意的是，启动的脚本是`./XXX`，不要写相对的路径`XXX/XXX`这种的，目录参考这样

```bash
ejabberd-22.10-1-linux-x64-src/
├── ejabberd
├── ejabberd-22.10
├── help.txt
└── setup
```
